📊 系统提示词层级架构

Oh-Myself 采用层叠式系统提示词设计，由 build_system_prompt() 函数动态构建，最终发送给 AI API。

Layer 1: Base Prompt (基础提示词)
始终存在

核心身份："你是 [user] 的长期成长助手"

语言：中文

内容：定义助手身份、工作原则、协作方式、回应风格、提问引导策略

特点：强调独立思考、反向思考、问题本质识别，避免情绪化反应

来源： _FOUNDATION_SYSTEM_PROMPT (第6-42行)
Layer 2: User Prompt (用户自定义提示词)
可选

配置方式：

  • CLI 参数: --system-prompt 或 -s
  • Settings 文件: system_prompt 字段

处理：通过 normalize_user_system_prompt() 函数处理，移除可能存在的旧版基础提示词

来源： Settings.system_prompt 或 CLI 参数
Layer 3: Model Profile + User Profile (模型/用户画像)
可选

Model Profile：AI 模型的行为指令，来自 {OHMYSELF_HOME}/memory/model_profile.md

User Profile：用户的持久化偏好/工作风格，来自 {OHMYSELF_HOME}/memory/user_profile.md

管理：通过 /memory 命令管理

来源： 本地文件系统动态加载
# Environment (环境信息)
始终追加

包含当前执行环境的详细信息：

  • 操作系统、Shell 类型
  • 当前工作目录 (CWD)
  • 当前日期
  • Python 版本、Git 信息
  • 是否为 Git 仓库
来源： _format_environment_section() (第48-66行)
# Goal Context (目标上下文)
目标模式下追加

内容：

  • 当前专注的目标主题
  • 目标进度、截止时间
  • 目标记忆 (AI 笔记、用户偏好、上下文)
  • 最近会话摘要
  • 当前模式指令
来源： GoalAgentContext.build_goal_context_prompt() (第89-121行)

🎯 设计哲学与特点

核心设计原则：层叠式、可配置、可继承、环境感知

1. 层叠式设计 (Layered Design)

  • 分离关注点：基础身份、用户自定义、个性化配置分离
  • 优先级明确：Layer 1 > Layer 2 > Layer 3，上层可覆盖下层
  • 可组合性：各层独立存在，可按需组合

2. 动态配置 (Dynamic Configuration)

  • 运行时加载：Layer 3 从文件系统动态加载
  • 多配置源：CLI 参数、Settings 文件、本地配置文件
  • 环境感知：自动注入环境信息，增强上下文感知

3. 继承一致性 (Inheritance Consistency)

  • 完整继承：子智能体获得父智能体的完整提示词
  • 行为一致：确保多智能体协作时行为一致
  • 上下文传递：目标上下文、环境信息完整传递

4. 模块化服务 (Modular Services)

  • 独立提示词：特定服务使用专用提示词
  • 单一职责：每个服务提示词专注于特定任务
  • 不污染主流程：服务提示词不影响主智能体行为
